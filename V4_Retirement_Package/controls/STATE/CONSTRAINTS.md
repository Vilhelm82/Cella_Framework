# Constraints ledger

## C-RET-001 no-automatic-authority-transfer

rule: Legacy claims, labels, proofs, tests, and certificates do not become Cella canon merely because they were organized or copied.

scope: All material in this retirement staging repository when considered for `/home/wlloyd/Cella Framework`.

not-scope: File preservation, indexing, historical comparison, and non-load-bearing research scouting.

why: Organization proves retrievability, not mathematical validity or implementation fitness.

displaced-by: A claim-specific Cella admission plus fresh in-repo derivation or re-verification.

## C-RET-002 preserve-origin-provenance

rule: Origin documents remain unchanged during the cleanup pass except for filesystem relocation.

scope: Historical reports, papers, campaign records, source, tests, audits, and handoffs moved in this reorganization.

not-scope: New staging metadata and generated inventory files under `STATE/`, plus new regional README files.

why: The next evaluator must be able to distinguish original evidence from later interpretation.

displaced-by: A claim-level consolidation that retains source paths, hashes, scope exclusions, and supersession links.
