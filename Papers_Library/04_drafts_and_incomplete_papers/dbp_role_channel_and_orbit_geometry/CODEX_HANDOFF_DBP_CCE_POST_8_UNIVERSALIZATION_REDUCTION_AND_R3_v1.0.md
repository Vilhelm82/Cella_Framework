### 1.1 Mandatory outcome vocabulary

Every investigated claim or expansion must end with exactly one of these dispositions:

1. `PROMOTED` -- proved, implemented where applicable, independently replayed, and admitted into the release theorem surface.
2. `ACTIVE_PROOF` -- not yet closed, but reduced to a named next exact object, lemma, witness, or computation. A vague “future work” label is not sufficient.
3. `BLOCKED` -- unable to proceed without a precisely named missing source, definition, authority, or external dependency. The affected claim is quarantined; independent lanes continue.
4. `REFUTED` -- disproved by an exact rank obstruction, counterexample, incompatible invariant, or other replayable argument. The refutation is a positive campaign result and must redirect the theorem statement.

`PARTIAL` may appear only as a summary label over individually classified subclaims. It may not hide an unnamed remainder.

### 1.2 Expansion-discovery rule

At the end of every work package:

1. scan the touched source closure and neighboring exact modules for generality already present but not exposed;
2. identify each fixed constant, finite list, special chart, route restriction, coefficient ring, or refusal boundary that functions only as an implementation fence;
3. distinguish a genuine mathematical hypothesis from a historical release floor;
4. promote any extension whose proof obligations can be closed exactly in the campaign;
5. classify every other extension using Section 1.1; and
6. record why preserving a restriction is mathematically necessary when it is retained.

Do not stop an independent work package because another package encounters a gap. Use local claim quarantine and continue all unaffected work.
